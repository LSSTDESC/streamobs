# Roman HLWAS Survey Files

This page describes the three Roman High Latitude Wide Area Survey (HLWAS)
configurations shipped with streamobs: `hlwas_wide`, `hlwas_medium`, and
`hlwas_all`.  All three reuse the selection-function tables (stellar completeness
and photometric-error curves) derived from the Roman-Rubin DC2 synthetic survey;
the only tier-specific data product is the F158 magnitude-limit map, which is
constructed from the official HLWAS exposure-time maps using the quasi-depth recipe
described below.

## Survey tiers

The Roman HLWAS community-defined survey comprises four nested tiers.  streamobs
currently supports three:

| Release | Footprint | F158 reported 5σ depth | Pixels (nside=1024) |
|---|---|---|---|
| `hlwas_wide`   | Wide tier only  (~2700 deg²) | 26.2 AB | ~1 028 570 |
| `hlwas_medium` | Medium tier only (~2400 deg²) | 26.4 AB | ~879 052   |
| `hlwas_all`    | All tiers stacked (wide + medium + deep + ultradeep) | 26.2 AB (wide median; wide dominates) | ~1 792 907 |

Reported depths are from the
[STScI HLWAS community-survey page](https://roman-docs.stsci.edu/roman-community-defined-surveys/high-latitude-wide-area-survey).

The deep and ultra-deep tiers (19 deg² and 4 deg² respectively) contribute a
negligible fraction of pixels to `hlwas_all` (<0.5%), so the map median stays at
the wide-tier reference depth of 26.2 AB.  Their pixels appear at greater depth
in the map (t >> 645 s → maglim >> 26.2) but do not shift the footprint-wide
statistics meaningfully.

## Exposure-time maps

Exposure-time maps (F158, healsparse `.hsp` format, nside=4096) are sourced from
the [spacetelescope/roman_notebooks](https://github.com/spacetelescope/roman_notebooks/tree/main/notebooks/footprint_visualization/aux_data)
repository (files `map_HLWAS-wide_F158.hsp`, `map_HLWAS-medium_F158.hsp`,
`map_HLWAS-all_F158.hsp`).  These encode the total on-sky integration time (in
seconds) at each sky position, accounting for the planned dither and pass
strategy.  The wide and medium tiers each have a median exposure time of
t_median = 645.1 s.

## Quasi-depth recipe

The magnitude-limit maps are computed from the exposure-time maps by the
quasi-depth recipe:

```
depth(pix) = reported_depth + 1.25 * log10( t(pix) / t_median )
```

where `t_median` is the median exposure time over the tier's footprint and
`reported_depth` is the STScI 5σ point-source total depth for that tier.
The 1.25 factor derives from the photon-noise scaling: S/N ∝ √t, so the
5σ limiting magnitude scales as 2.5 × 0.5 × log10(t) = 1.25 × log10(t).

### DC2-relative convention

The delta_mag axes of the DC2-derived completeness and photometric-error tables
are keyed to the F158 map median of the DC2 truth-anchored depth map
(26.38 AB; see the [Roman DC2 Survey Files](roman_dc2.md) page for the full
derivation).  Accordingly, the HLWAS maglim maps must be expressed in the
**same truth-anchored S/N=5 convention** — scaled relative to the DC2 depth —
for the DC2-derived tables to apply unchanged.

The `reported_depth` values used as anchors in the recipe (26.2 AB for wide/all,
26.4 AB for medium) are the STScI official point-source depths.  These are
**not the same** as the DC2 truth-anchored median (26.38 AB): the STScI values
assume optimal PSF photometry, whereas the DC2 depth is calibrated to the
catalog's `mag_auto` in true-magnitude space.  The ~0.2 mag offset is the
`mag_auto` aperture loss absorbed into the DC2 convention.  Using the HLWAS
reported depths as the `reported_depth` anchor implicitly shifts the
completeness and photo-error curves by this offset relative to the DC2
reference depth, which is the physically correct thing to do: the DC2 tables
characterise the selection function in `delta_mag = mag - maglim` space, and
the HLWAS maglim maps are constructed so that `delta_mag = 0` corresponds to
the same true-magnitude S/N limit.

A future upgrade will replace this recipe with ETC-derived depth maps (or
truth-anchored maps from a dedicated HLWAS-depth simulation), which would
remove this reliance on the quasi-depth approximation.

### Measured map medians

| Release | t_median (s) | reported_depth (AB) | measured map median (AB) |
|---|---|---|---|
| `hlwas_wide`   | 645.1 | 26.2 | 26.21 |
| `hlwas_medium` | 645.1 | 26.4 | 26.41 |
| `hlwas_all`    | 645.1 | 26.2 | 26.21 |

The maps are written at nside=1024 (RING ordering, float32) to match the DC2
maglim maps.  They are generated by `scripts/roman/build_hlwas_maglim_maps.py`
and written to `data/surveys/roman_hlwas_<tier>/` (gitignored).

## Selection-function tables

The completeness and photometric-error tables are **identical** to those used for
`roman_dc2` and live under `data/surveys/roman_dc2/`:

| File | Description |
|---|---|
| `roman_stellar_efficiency_cutf158.csv` | F158 stellar detection + classification efficiency vs delta_mag |
| `roman_photoerror_f158_catalog.csv` | Median reported SExtractor magerr vs delta_mag (drives S/N cut) |
| `roman_photoerror_f158.csv` | Truth-based scatter of (obs − true) vs delta_mag (drives noise draw) |

The files are symlinked from each tier's data directory to `roman_dc2/` by the
build script, so the survey loader finds them at the default path.  See the
[Roman DC2 Survey Files](roman_dc2.md) page for the full derivation of these
tables.

## Configuration

The three tiers are each configured by a YAML file in `config/surveys/`:

- `config/surveys/roman_hlwas_wide.yaml` — release `hlwas_wide`
- `config/surveys/roman_hlwas_medium.yaml` — release `hlwas_medium`
- `config/surveys/roman_hlwas_all.yaml` — release `hlwas_all`

All use F158 only (F106/F129 maps are not yet constructed for the HLWAS
footprint; F184 is excluded per the DC2 documentation).  Extinction coefficients
and saturation threshold are copied from `roman_dc2.yaml` (same instrument).

The `delta_saturation` parameter (= saturation − map median) differs across tiers
because it is keyed to each tier's measured map median:

| Release | map_median (AB) | delta_saturation |
|---|---|---|
| `hlwas_wide`   | 26.21 | −9.21 |
| `hlwas_medium` | 26.41 | −9.41 |
| `hlwas_all`    | 26.21 | −9.21 |

## Using the surveys in streamobs

```python
from streamobs.surveys import Survey

# Load individual tiers
survey_wide   = Survey.load('roman', release='hlwas_wide')
survey_medium = Survey.load('roman', release='hlwas_medium')
survey_all    = Survey.load('roman', release='hlwas_all')

# Access the F158 maglim map
maglim_map = survey_wide.get_maglim('f158')

# Get completeness and photo error at a pixel
pix = 123456
maglim = survey_wide.get_maglim('f158', pixel=pix)
eff   = survey_wide.get_completeness('f158', mag=25.0, maglim=maglim)
err   = survey_wide.get_photo_error('f158', magnitude=25.0, maglim=maglim)
```

The column namespace for each release follows the `{name}_{release}` convention:
`roman_hlwas_wide`, `roman_hlwas_medium`, `roman_hlwas_all`.

## Regenerating the maps

```bash
conda activate streamobs
python scripts/roman/build_hlwas_maglim_maps.py
```

This reads the healsparse exposure-time maps from
`~/software/roman_notebooks/notebooks/footprint_visualization/aux_data/`,
applies the quasi-depth recipe, writes the three nside=1024 maglim maps, and
creates symlinks to the DC2 CSV files in each tier's data directory.

## Caveats

- The quasi-depth recipe assumes photon-noise-limited exposures and a uniform
  PSF.  Real HLWAS depths will vary due to detector gaps, background fluctuations,
  and read-noise contributions in the shortest exposures.
- The DC2-derived completeness and photo-error tables characterise the DC2
  reference simulation (~26.9 AB deep, ~20 deg²).  Their use for the HLWAS tiers
  relies on the assumption that the selection function depends on magnitude
  only through `delta_mag = mag − maglim`, i.e. that depth variations translate
  to a uniform shift along the magnitude axis.
- The `hlwas_all` map includes deep and ultra-deep tier pixels (t up to
  ~23 600 s; maglim up to ~29.5 AB locally) where the selection function from
  the shallow DC2 simulation may not be reliable.
- ETC-derived or truth-anchored depth maps for the real HLWAS are a future
  upgrade that will supersede this quasi-depth approach.
